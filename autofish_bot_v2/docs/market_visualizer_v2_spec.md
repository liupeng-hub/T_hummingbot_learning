# 行情可视化系统增强版规格文档

## 1. 项目概述

将现有的行情可视化功能升级为完整的 Web 应用系统，支持：
- 测试用例管理和选择
- 数据库持久化存储
- 交互式 K 线图表展示
- Web 端触发行情测试实例
- 测试结果数据管理

## 3. 数据库设计

### 3.1 数据库文件位置
- 路径: `market_visualizer_out/market_visualizer.db`
- 类型: SQLite
- **注意**: K线数据使用现有的 `kline_cache/klines.db`，本数据库不存储K线缓存

### 3.2 表结构设计

#### 3.2.1 测试用例表 (test_cases)
```sql
CREATE TABLE test_cases (
    id TEXT PRIMARY KEY,                     -- UUID
    name TEXT NOT NULL,                    -- 测试名称
    symbol TEXT NOT NULL,                  -- 交易对
    interval TEXT NOT NULL DEFAULT '1d',   -- K线周期
    start_date TEXT NOT NULL,              -- 开始日期
    end_date TEXT NOT NULL,                -- 结束日期
    algorithm TEXT NOT NULL,               -- 算法名称
    algorithm_config TEXT,                 -- 算法参数 (JSON)
    description TEXT,                      -- 描述
    created_at TEXT NOT NULL,              -- 创建时间
    updated_at TEXT NOT NULL,              -- 更新时间
    status TEXT DEFAULT 'pending',         -- 状态: pending/running/completed/failed
    UNIQUE(symbol, interval, start_date, end_date, algorithm, algorithm_config)
);
```

#### 3.2.2 测试结果表 (test_results)
```sql
CREATE TABLE test_results (
    id TEXT PRIMARY KEY,                     -- UUID
    test_case_id TEXT NOT NULL,            -- 关联测试用例 (UUID)
    total_days INTEGER,                    -- 总天数
    ranging_days INTEGER,                  -- 震荡天数
    trending_up_days INTEGER,              -- 上涨天数
    trending_down_days INTEGER,            -- 下跌天数
    ranging_count INTEGER,                 -- 震荡区间数
    trending_up_count INTEGER,             -- 上涨区间数
    trending_down_count INTEGER,           -- 下跌区间数
    status_ranges TEXT,                    -- 区间数据 (JSON)
    executed_at TEXT NOT NULL,             -- 执行时间
    duration_ms INTEGER,                   -- 执行耗时
    FOREIGN KEY (test_case_id) REFERENCES test_cases(id)
);
```

#### 3.2.3 每日状态表 (daily_statuses)
```sql
CREATE TABLE daily_statuses (
    id TEXT PRIMARY KEY,                     -- UUID
    test_result_id TEXT NOT NULL,          -- 关联测试结果 (UUID)
    date TEXT NOT NULL,                    -- 日期
    status TEXT NOT NULL,                  -- 状态: ranging/trending_up/trending_down
    confidence REAL,                       -- 置信度
    reason TEXT,                           -- 原因
    open_price REAL,                       -- 开盘价
    close_price REAL,                      -- 收盘价
    high_price REAL,                       -- 最高价
    low_price REAL,                        -- 最低价
    volume REAL,                           -- 成交量
    FOREIGN KEY (test_result_id) REFERENCES test_results(id)
);
CREATE INDEX idx_daily_statuses_result ON daily_statuses(test_result_id);
CREATE INDEX idx_daily_statuses_date ON daily_statuses(date);
```

### 3.3 外部数据源

#### 3.3.1 K线数据
- **数据源**: `kline_cache/klines.db` (现有数据库)
- **访问方式**: 通过 `binance_kline_fetcher.KlineFetcher` 模块
- **说明**: 本系统不重复存储K线数据，直接使用现有缓存

## 4. Web 界面设计

### 4.1 响应式布局设计

#### 4.1.1 设计原则
- **自适应宽度**: 页面和图表根据窗口宽度自动调整
- **带鱼屏优化**: 充分利用超宽屏幕空间
- **最小宽度**: 支持最小 1200px 宽度
- **最大宽度**: 无上限，自动填充

#### 4.1.2 布局断点

| 断点 | 宽度范围 | 布局方式 |
|------|----------|----------|
| 超宽屏 | >= 1920px | 左侧列表 300px，右侧图表自适应 |
| 宽屏 | 1440px - 1919px | 左侧列表 280px，右侧图表自适应 |
| 标准 | 1200px - 1439px | 左侧列表 250px，右侧图表自适应 |
| 窄屏 | < 1200px | 不支持，显示提示 |

### 4.2 图表自适应实现

#### 4.2.1 Lightweight Charts 自适应配置

```javascript
// 创建图表时使用 autoSize
const chart = createChart(container, {
    autoSize: true,  // 自动调整大小
    layout: {
        background: { color: '#ffffff' },
        textColor: '#333',
    },
    crosshair: {
        mode: CrosshairMode.Normal,
    },
    // 响应式高度设置
    height: Math.max(500, window.innerHeight * 0.5),
});

// 监听窗口大小变化
window.addEventListener('resize', () => {
    chart.applyOptions({
        height: Math.max(500, window.innerHeight * 0.5),
    });
    chart.timeScale().fitContent();
});
```

#### 4.4.4 对比功能特性

1. **图表同步**
   - 所有图表的时间范围同步
   - 缩放/拖拽一个图表，其他图表同步移动
   - 十字光标同步显示

2. **统计对比**
   - 并排显示各测试的统计数据
   - 高亮显示差异项
   - 支持导出对比报告

3. **对比限制**
   - 最少选择 2 个测试
   - 最多选择 4 个测试
   - 建议对比相同交易对、相同时间范围的测试

// 图表同步管理器
class ChartSyncManager {
    constructor() {
        this.charts = [];
        this.isSyncing = false;
    }
    
    addChart(chart) {
        this.charts.push(chart);
        
        // 监听可见范围变化
        chart.timeScale().subscribeVisibleTimeRangeChange((range) => {
            if (this.isSyncing) return;
            this.syncRange(chart, range);
        });
    }
    
    syncRange(sourceChart, range) {
        this.isSyncing = true;
        
        this.charts.forEach(chart => {
            if (chart !== sourceChart) {
                chart.timeScale().setVisibleRange(range);
            }
        });
        
        this.isSyncing = false;
    }
    
    // 同步十字光标

            }
        });
    }
}

## 6. K线图表插件选型

### 6.1 候选方案对比

| 插件 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **Lightweight Charts** | TradingView出品、轻量、性能好、专业K线 | 功能相对简单 | ⭐⭐⭐⭐⭐ |
| ECharts | 功能全面、中文文档好 | 体积大、K线样式一般 | ⭐⭐⭐⭐ |
| TradingView Widget | 最专业、功能最强 | 需要付费、依赖网络 | ⭐⭐⭐ |
| D3.js | 灵活度最高 | 开发成本高 | ⭐⭐⭐ |
