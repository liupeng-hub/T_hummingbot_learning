# 资金显示优化计划

## 目标
优化入场资金和入场总资金的显示，使它们分别表示：
1. **入场资金**：每一轮次中使用的资金，用于计算每个 level 的订单金额
2. **入场总资金**：每次交易盈利或亏损后的总资金值

## 现状分析
- 当前逻辑：所有订单的 `entry_capital` 和 `entry_total_capital` 都相同
- 对于固定资金策略，都等于初始资金
- 对于复利策略，都等于当前交易资金

## 实现计划

### [ ] 任务 1：修改资金池逻辑
- **优先级**：P1
- **依赖**：None
- **描述**：
  - 跟踪每轮交易的盈利/亏损
  - 在每轮交易结束后更新总资金
  - 为下一轮交易准备新的资金值
- **成功标准**：
  - 资金池能够正确跟踪每轮交易的盈利/亏损
  - 总资金能够在每轮交易后更新
- **测试要求**：
  - `programmatic` TR-1.1：验证资金池在交易后正确更新
  - `programmatic` TR-1.2：验证总资金计算正确

### [ ] 任务 2：修改订单资金分配逻辑
- **优先级**：P1
- **依赖**：任务 1
- **描述**：
  - A1 订单使用当前总资金作为入场资金
  - A2/A3/A4 订单继承 A1 的入场资金
  - 确保每个 level 的订单金额计算正确
- **成功标准**：
  - A1 订单使用最新的总资金
  - A2/A3/A4 订单使用与 A1 相同的资金
  - 订单金额计算正确
- **测试要求**：
  - `programmatic` TR-2.1：验证 A1 订单使用最新总资金
  - `programmatic` TR-2.2：验证 A2/A3/A4 订单继承 A1 的资金

### [ ] 任务 3：修改交易记录逻辑
- **优先级**：P1
- **依赖**：任务 2
- **描述**：
  - 在交易记录中保存正确的入场资金和入场总资金
  - 确保入场总资金反映交易时的实际总资金
- **成功标准**：
  - 交易记录中的入场资金正确
  - 交易记录中的入场总资金正确
- **测试要求**：
  - `programmatic` TR-3.1：验证交易记录中的资金值正确
  - `human-judgment` TR-3.2：验证资金值的显示逻辑合理

### [ ] 任务 4：修改 Web 界面显示
- **优先级**：P2
- **依赖**：任务 3
- **描述**：
  - 更新 Web 界面，确保正确显示入场资金和入场总资金
  - 确保数据可视化正确反映资金变化
- **成功标准**：
  - Web 界面显示正确的资金值
  - 数据可视化能够反映资金变化
- **测试要求**：
  - `human-judgment` TR-4.1：验证 Web 界面显示正确
  - `human-judgment` TR-4.2：验证数据可视化合理

## 技术实现要点

### 资金池修改
1. 尊重资金池策略的实现逻辑
2. 对于 `fuli` 策略，跟踪每轮交易的盈利/亏损，在每轮交易结束后更新总资金
3. 对于其他策略，保持其原有逻辑

### 订单资金分配
1. 根据资金池策略确定 A1 订单的入场资金
   - `fuli` 策略：使用最新总资金
   - 其他策略：使用各自的资金逻辑
2. A2/A3/A4 订单继承 A1 的入场资金，确保资金值在订单链中保持一致
3. 确保每个 level 的订单金额计算正确

### 交易记录
1. 确保交易记录中的资金值正确反映交易时的状态
2. 保存每轮交易的资金变化
3. 确保入场总资金反映交易时的实际总资金

### Web 界面
1. 更新数据显示逻辑，确保正确显示不同策略的资金值
2. 确保资金变化可视化

## 预期效果
- 入场资金：显示每一轮次中使用的资金，用于计算订单金额
- 入场总资金：显示每次交易盈利或亏损后的总资金值
- 资金变化能够被正确跟踪和显示
