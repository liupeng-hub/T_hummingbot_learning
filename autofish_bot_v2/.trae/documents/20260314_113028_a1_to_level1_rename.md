# A1 代码命名规范化方案

## 背景

"A1" 是用户习惯的说法（第一层订单），但在代码中使用 "a1" 不够规范。需要改为与系统现有命名一致的名称。

## 系统现有命名分析

### 订单属性
- `order.level` - 订单层级属性（1, 2, 3, ...）
- `order.entry_price` - 入场价格
- `max_entries` - 最大层级数

### 日志/通知显示
- `A{level}` - 如 A1, A2, A3（用户可见，保持不变）

### 命名建议

基于系统现有命名模式，建议使用 **`first_entry`** 或 **`level1`**：

| 候选名称 | 优点 | 缺点 |
|----------|------|------|
| `first_entry` | 与 `entry_price` 一致，语义清晰 | 较长 |
| `level1` | 与 `order.level` 一致，简洁 | 数字开头不太规范 |
| `initial_entry` | 语义明确 | 较长 |
| `entry_level_1` | 完整描述 | 过长 |

**推荐方案**：使用 **`first_entry`**，理由：
1. 与系统现有 `entry_price`、`entry_order` 命名风格一致
2. 语义清晰，表示"第一笔入场订单"
3. 符合 Python 命名规范

## 分析结果

### 需要修改的地方

| 类型 | 当前命名 | 建议修改 | 原因 |
|------|----------|----------|------|
| 配置参数 | `a1_timeout_minutes` | **不改** | 用户配置接口，保持兼容 |
| 方法名 | `get_pending_a1()` | `get_pending_first_entry()` | 内部方法 |
| 方法名 | `check_a1_timeout()` | `check_first_entry_timeout()` | 内部方法 |
| 方法名 | `_check_and_handle_a1_timeout()` | `_check_and_handle_first_entry_timeout()` | 内部方法 |
| 函数名 | `notify_a1_timeout_refresh()` | `notify_first_entry_timeout_refresh()` | 内部函数 |
| 变量名 | `timeout_a1` | `timeout_first_entry` | 内部变量 |
| 日志/打印 | "A1 超时" | **不改** | 用户可见，保持可读性 |

### 不需要修改的地方

- 用户可见的配置参数：`a1_timeout_minutes`
- 用户可见的日志/打印输出：保持 "A1" 便于理解
- 订单属性 `order.level`：已经是规范命名

## 注意事项

1. **保持向后兼容**：配置参数 `a1_timeout_minutes` 不变，用户配置无需修改
2. **用户可见信息不变**：日志、打印输出、微信通知中的 "A1" 保持不变
3. **仅修改内部代码**：方法名、变量名等内部实现
