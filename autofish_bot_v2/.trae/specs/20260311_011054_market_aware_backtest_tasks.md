# 行情分析与回测结合 - 任务列表

## 阶段一：基础架构

### Task 1.1: 创建行情感知回测引擎基类
- **优先级**: 高
- **依赖**: 无
- **描述**: 创建 `MarketAwareBacktestEngine` 类，继承自 `BacktestEngine`
- **输出**: `market_aware_backtest.py`

### Task 1.2: 实现行情状态数据结构
- **优先级**: 高
- **依赖**: 无
- **描述**: 创建 `MarketStatusEvent` 数据类，用于记录行情状态变化
- **输出**: 在 `market_aware_backtest.py` 中添加

### Task 1.3: 实现多周期 K线获取
- **优先级**: 高
- **依赖**: Task 1.1
- **描述**: 在回测引擎中同时获取 1m 和 1d K线数据
- **输出**: `_fetch_multi_interval_klines()` 方法

## 阶段二：行情判断集成

### Task 2.1: 实现行情状态检查逻辑
- **优先级**: 高
- **依赖**: Task 1.3
- **描述**: 实现每天第一根 1m K线时更新行情判断的逻辑
- **输出**: `_check_market_status()` 方法

### Task 2.2: 实现状态切换检测
- **优先级**: 高
- **依赖**: Task 2.1
- **描述**: 检测行情状态变化，触发相应动作
- **输出**: `_detect_status_change()` 方法

### Task 2.3: 实现交易控制逻辑
- **优先级**: 高
- **依赖**: Task 2.2
- **描述**: 
  - 趋势行情：平仓所有订单，停止交易
  - 震荡行情：开始交易，创建 A1 订单
- **输出**: `_on_market_status_change()` 方法

## 阶段三：订单管理扩展

### Task 3.1: 实现强制平仓方法
- **优先级**: 高
- **依赖**: Task 2.3
- **描述**: 实现按当前价格平仓所有已成交订单的方法
- **输出**: `_close_all_positions()` 方法

### Task 3.2: 重写 K线处理逻辑
- **优先级**: 高
- **依赖**: Task 3.1
- **描述**: 重写 `_on_kline()` 方法，集成行情判断和交易控制
- **输出**: 修改后的 `_on_kline()` 方法

### Task 3.3: 实现交易状态管理
- **优先级**: 中
- **依赖**: Task 3.2
- **描述**: 维护 `trading_enabled` 状态，控制是否允许交易
- **输出**: 状态管理逻辑

## 阶段四：报告扩展

### Task 4.1: 扩展回测结果数据结构
- **优先级**: 中
- **依赖**: Task 2.3
- **描述**: 在 `results` 中添加行情相关字段
- **输出**: 扩展的 `results` 字典

### Task 4.2: 实现行情统计计算
- **优先级**: 中
- **依赖**: Task 4.1
- **描述**: 计算震荡/趋势时间占比、交易时段等统计
- **输出**: `_calculate_market_statistics()` 方法

### Task 4.3: 扩展报告生成
- **优先级**: 中
- **依赖**: Task 4.2
- **描述**: 在回测报告中添加"行情分析"章节
- **输出**: 修改后的 `save_report()` 方法

### Task 4.4: 扩展历史记录
- **优先级**: 低
- **依赖**: Task 4.3
- **描述**: 在历史记录中添加行情相关字段
- **输出**: 修改后的 `save_history()` 方法

## 阶段五：命令行接口

### Task 5.1: 添加命令行参数
- **优先级**: 中
- **依赖**: Task 1.1
- **描述**: 添加 `--market-aware`、`--market-interval`、`--market-algorithm` 等参数
- **输出**: 修改后的 `main()` 函数

### Task 5.2: 实现参数解析
- **优先级**: 中
- **依赖**: Task 5.1
- **描述**: 解析命令行参数，构建市场配置
- **输出**: 参数解析逻辑

## 阶段六：测试与验证

### Task 6.1: 编写单元测试
- **优先级**: 中
- **依赖**: 阶段一~三
- **描述**: 测试行情判断、状态切换、交易控制等核心逻辑
- **输出**: `test_market_aware_backtest.py`

### Task 6.2: 运行对比回测
- **优先级**: 高
- **依赖**: 阶段一~五
- **描述**: 
  - 运行无行情控制的回测
  - 运行有行情控制的回测
  - 对比结果
- **输出**: 对比报告

### Task 6.3: 验证报告生成
- **优先级**: 中
- **依赖**: Task 6.2
- **描述**: 验证报告中的行情分析章节是否正确
- **输出**: 验证结果
