# 测试管理系统数据库化重构 Spec

## Why

当前测试计划以 JSON 文件形式存储，存在以下问题：
1. **数据分散**：测试计划、参数、结果分散在文件和数据库中
2. **管理不便**：无法通过 Web 界面完整管理测试计划生命周期
3. **一致性差**：文件和数据库数据可能不一致
4. **功能不完整**：缺少完整的测试管理功能（对比、历史汇总等）

通过将所有测试数据统一存储到数据库，可以实现：
- 统一的测试数据管理
- 完整的测试生命周期管理
- 便捷的数据导出和报告生成

## What Changes

- **BREAKING**: 移除文件形式的测试计划存储，统一使用数据库
- 清空现有数据库记录，保留表结构
- 将现有测试计划文件导入数据库
- 完善测试管理 API 接口
- 设计并实现测试历史汇总功能
- 设计并实现测试对比功能
- 设计 MD 报告模板

## Impact

- Affected specs: test_manager.py, database/test_results_db.py
- Affected code: 
  - 新增 test_history 目录
  - 新增 test_comparison 目录
  - 移除 test_results 目录（功能已由 test_report 覆盖）
- Affected data: out/test_plans/active/*.json 将导入数据库

#### 对比逻辑说明

**最优结果判定规则**：
1. 计算每个结果的加权得分
2. 权重配置：胜率(30%) + 收益率(40%) + 超额收益(30%)
3. 得分最高者为综合最优

**参数差异标记**：
- 所有结果值相同：无标记
- 存在不同值：标记 ⚠️

**指标最优标记**：
- 单项指标最优：✅
- 多个结果并列最优：都标记 ✅

## MODIFIED Requirements

### Requirement: 测试计划 API

修改测试计划相关 API：
- `GET /api/test-plans` - 从数据库获取测试计划列表
- `POST /api/test-plans` - 创建测试计划到数据库
- `GET /api/test-plans/{plan_id}` - 从数据库获取测试计划详情
- `PUT /api/test-plans/{plan_id}` - 更新数据库中的测试计划
- `DELETE /api/test-plans/{plan_id}` - 删除数据库中的测试计划
- `POST /api/test-plans/{plan_id}/run` - 执行测试计划
- `GET /api/test-plans/{plan_id}/export` - 导出测试计划（MD/JSON）

### Requirement: 测试历史 API

新增测试历史相关 API：
- `GET /api/history` - 获取测试历史汇总
- `GET /api/history/{symbol}` - 获取指定标的的测试历史
- `GET /api/history/export` - 导出测试历史（MD/CSV）

### Requirement: 测试对比 API

新增测试对比相关 API：
- `POST /api/compare` - 对比多个测试结果
- `GET /api/compare/{comparison_id}` - 获取对比结果
- `POST /api/compare/export` - 导出对比报告（MD）
