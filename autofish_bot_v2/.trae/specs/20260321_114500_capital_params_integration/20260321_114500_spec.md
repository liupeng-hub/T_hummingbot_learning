# 资金池参数整合与扩展策略集成 Spec

## 背景

当前资金池参数（capital_mode, initial_capital, withdrawal_strategy）是独立的 CLI 参数，与其他参数（amplitude-params, market-params, entry-params）的组织方式不一致。需要将资金池参数整合为单一 JSON 参数，并集成到扩展策略文件和测试管理系统中。

## 变更内容

- 将资金池参数整合为 `--capital-params` 参数（JSON 格式）
- 在 `autofish_extern_strategy.json` 中添加资金池策略配置
- 在 test_manager 中增加资金池参数支持
- 数据库中增加资金池配置存储
- 资金数据与测试用例关联（支持级联删除）

## 影响范围

- 影响代码:
  - `binance_backtest.py` (CLI 参数整合)
  - `out/autofish/autofish_extern_strategy.json` (扩展策略配置)
  - `test_manager.py` (资金池参数支持)
  - `database/test_results_db.py` (数据库表结构)
  - `web/test_results/` (前端页面，如需要)

## 新增需求

### 需求: CLI 参数整合

系统 SHALL 将资金池参数整合为单一 JSON 参数：

#### 参数格式

```bash
# 固定模式
--capital-params '{"mode": "fixed"}'

# 递进模式（保守策略）
--capital-params '{"mode": "progressive", "initial_capital": 10000, "strategy": "conservative"}'

# 递进模式（自定义参数）
--capital-params '{
  "mode": "progressive",
  "initial_capital": 10000,
  "strategy": "custom",
  "withdrawal_threshold": 2.0,
  "withdrawal_retain": 1.5,
  "liquidation_threshold": 0.1
}'
```

#### 参数结构

```json
{
  "mode": "fixed | progressive",
  "initial_capital": 10000,
  "strategy": "conservative | aggressive | very_conservative | custom",
  "withdrawal_threshold": 2.0,
  "withdrawal_retain": 1.5,
  "liquidation_threshold": 0.1
}
```

#### 场景: CLI 参数解析成功

- **当** 用户指定 `--capital-params '{"mode": "progressive", "initial_capital": 10000}'`
- **则** 系统正确解析参数
- **且** 使用递进模式，初始资金 10000 USDT

#### capital_statistics 表说明

`capital_statistics` 表存储资金池统计数据（共 18 个统计字段）：

| 字段名 | 类型 | 说明 |

| result_id | TEXT | 关联的测试结果 ID（UNIQUE） |
| initial_capital | REAL | 初始资金 |
| final_capital | REAL | 最终资金（trading_capital + profit_pool） |
| trading_capital | REAL | 当前交易资金 |
| profit_pool | REAL | 利润池（锁定的利润） |
| total_return | REAL | 总收益率（%） |
| total_profit | REAL | 累计盈利 |
| total_loss | REAL | 累计亏损 |
| max_capital | REAL | 历史最高资金 |
| max_drawdown | REAL | 最大回撤（%） |
| withdrawal_count | INTEGER | 提现次数 |
| total_withdrawal | REAL | 累计提现金额 |
| liquidation_count | INTEGER | 爆仓次数 |
| total_trades | INTEGER | 总交易次数 |
| profit_trades | INTEGER | 盈利交易次数 |
| loss_trades | INTEGER | 亏损交易次数 |
| avg_profit | REAL | 平均盈利 |
| avg_loss | REAL | 平均亏损 |
| win_rate | REAL | 胜率（%） |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

#### capital_history 表说明

`capital_history` 表存储每次资金变化的详细记录：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INTEGER | 主键 |
| result_id | TEXT | 关联的测试结果 ID |
| statistics_id | INTEGER | 关联的统计数据 ID |
| timestamp | TEXT | 时间戳 |
| old_capital | REAL | 变化前资金 |
| new_capital | REAL | 变化后资金 |
| profit | REAL | 本次盈亏 |
| event_type | TEXT | 事件类型（trade/withdrawal/liquidation） |
| created_at | DATETIME | 创建时间 |

**用途**: 
- 记录每次资金变化的详细历史
- 支持资金变化曲线可视化
- 支持详细的资金管理分析

### 需求: 数据关联与级联删除

系统 SHALL 正确处理资金数据的关联和删除：

#### 数据关联

```
test_plans (capital_params)
    ↓
test_results (capital_params)
    ↓
capital_statistics (result_id → test_results.result_id)
    ↓
capital_history (result_id → test_results.result_id, statistics_id → capital_statistics.id)
    ↓
trade_details (通过 result_id 关联)
```

#### 级联删除规则

1. 删除 test_plan 时：
   - 删除关联的 test_results
   - 级联删除 capital_statistics（通过 test_results.result_id）
   - 级联删除 capital_history（通过 test_results.result_id 和 capital_statistics.id）
   - 删除关联的 trade_details

2. 删除 test_result 时：
   - 级联删除 capital_statistics（通过 result_id）
   - 级联删除 capital_history（通过 result_id 和 statistics_id）
   - 删除关联的 trade_details

3. 删除 capital_statistics 时：
   - 级联删除 capital_history（通过 statistics_id）

#### 场景: 级联删除成功

- **当** 用户删除测试计划 ID=1
- **则** 系统删除 test_plans 中 ID=1 的记录
- **且** 删除 test_results 中 plan_id=1 的所有记录
- **且** 删除 capital_statistics 中关联的所有记录
- **且** 删除 capital_history 中关联的所有记录
- **且** 删除 trade_details 中关联的所有记录
- **且** 数据完整性保持一致

### 需求: MarketAwareBacktestEngine 参数更新

`MarketAwareBacktestEngine` SHALL 支持新的参数格式：

```python
def __init__(
    self,
    amplitude: Dict,
    market: Dict,
    entry: Dict,
    timeout: Dict,
    capital: Dict = None,
):
    if capital:
        self.capital_mode = capital.get('mode', 'fixed')
        self.initial_capital = Decimal(str(capital.get('initial_capital', 10000)))
        strategy = capital.get('strategy', 'conservative')
        
        if self.capital_mode == 'progressive':
            self.capital_pool = Autofish_CapitalPool(self.initial_capital)
            if strategy == 'custom':
                self.capital_pool.withdrawal_threshold = Decimal(str(capital.get('withdrawal_threshold', 2.0)))
                self.capital_pool.withdrawal_retain = Decimal(str(capital.get('withdrawal_retain', 1.5)))
                self.capital_pool.liquidation_threshold = Decimal(str(capital.get('liquidation_threshold', 0.1)))
            else:
                self.capital_pool.set_strategy(strategy)
```
