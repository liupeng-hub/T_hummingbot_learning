# autofish_bot_v2 算法与回测系统梳理文档

## 背景
autofish_bot_v2 项目经过多次迭代开发，积累了大量功能代码。需要系统性地梳理交易算法原理、回测程序实现逻辑，评估回测实现的正确性，特别是复利资金算法的准确性，以便于后续维护和优化。

## 变更内容
- 整理交易算法原理文档
- 分析回测程序执行流程
- 绘制回测流程图
- 评估复利资金算法正确性
- 输出完整的技术文档到 docs 目录

## 影响范围
- 涉及代码: autofish_core.py, binance_backtest.py, test_manager.py, market_status_visualizer.py, binance_kline_fetcher.py
- 新增文档: docs/ 目录下的技术文档

## 新增需求

### 需求：交易算法原理文档
系统应提供完整的交易算法原理说明文档，包括：
- 市场状态检测算法
- 入场条件判断逻辑
- 出场条件判断逻辑
- 订单组管理机制

#### 场景：算法原理说明
- **当** 开发者阅读文档时
- **则** 能够清晰理解交易算法的核心逻辑和设计思路

### 需求：回测程序执行流程文档
系统应提供回测程序的完整执行流程说明，包括：
- 数据获取流程
- K线处理流程
- 交易执行流程
- 资金管理流程
- 结果统计流程

#### 场景：流程图展示
- **当** 开发者查看流程图时
- **则** 能够直观理解回测程序的执行顺序和数据流向

### 需求：复利资金算法评估
系统应对复利资金算法进行详细评估，包括：
- 资金池管理逻辑分析
- 复利计算公式验证
- 提现机制分析
- 爆仓处理逻辑分析

#### 场景：算法正确性验证
- **当** 开发者阅读评估报告时
- **则** 能够判断当前复利实现是否符合预期设计

### 需求：辅助模块分析文档
系统应提供辅助模块的分析文档，包括：
- test_manager.py 测试管理模块
- market_status_visualizer.py 行情可视化模块
- binance_kline_fetcher.py K线数据获取模块

#### 场景：模块功能说明
- **当** 开发者阅读文档时
- **则** 能够理解各辅助模块的功能和与主系统的交互关系
