# 测试管理系统数据库重构 Spec

## Why

基于 `market_aware_backtest` 的 CLI 能力，对 `test_manager` 进行重构，当前测试数据分散在文件和数据库中，管理不便，需要统一数据存储结构，完善测试管理功能。

## What Changes

- **BREAKING**: 重构测试管理数据库结构
- 新增测试用例表（test_cases）
- 新增测试结果表（test_results）
- 新增交易详情表（trade_details）
- 完善测试管理 API 接口
- 设计测试历史汇总功能
- 设计测试对比功能
- 设计 MD 报告模板

## Impact

- Affected specs: test_manager.py, database/test_results_db.py
- Affected code:
  - 新增 test_history 目录
  - 新增 test_comparison 目录
  - 移除 test_results 目录（功能已由 test_report 覆盖）
- Affected data: out/test_plans/active/*.json 将导入数据库

## ADDED Requirements

### Requirement: 数据表设计

系统 SHALL 使用以下数据表结构：

#### 1. 测试用例表 (test_cases)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | TEXT (UUID) | 主键 |
| name | TEXT | 测试用例名称 |
| test_type | TEXT | 测试类型 (backtest/market_aware/visualizer/optimization) |
| symbol | TEXT | 交易标的 |
| date_range | TEXT | 时间范围 (yyyymmdd-yyyymmdd) |
| amplitude | TEXT (JSON) | 振幅配置 |
| market | TEXT (JSON) | 行情算法配置 |
| entry | TEXT (JSON) | 入场策略配置 |
| timeout | TEXT (JSON) | 超时配置 |
| description | TEXT | 描述 |
| status | TEXT | 状态 (draft/active/completed/archived) |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

#### 2. 测试结果表 (test_results)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | TEXT (UUID) | 主键 |
| case_id | TEXT | 关联测试用例ID |
| execution_id | TEXT | 执行ID |
| symbol | TEXT | 交易标的 |
| interval | TEXT | K线周期 |
| start_time | DATETIME | 开始时间 |
| end_time | DATETIME | 结束时间 |
| klines_count | INTEGER | K线数量 |
| total_trades | INTEGER | 总交易数 |
| win_trades | INTEGER | 盈利交易数 |
| loss_trades | INTEGER | 亏损交易数 |
| win_rate | REAL | 胜率 |
| total_profit | REAL | 总盈利 |
| total_loss | REAL | 总亏损 |
| net_profit | REAL | 净收益 |
| roi | REAL | 收益率 |
| price_change | REAL | 价格变化 |
| excess_return | REAL | 超额收益 |
| profit_factor | REAL | 盈亏比 |
| sharpe_ratio | REAL | 夏普比率 |
| max_profit_trade | REAL | 最大单笔盈利 |
| max_loss_trade | REAL | 最大单笔亏损 |
| trading_time_ratio | REAL | 交易时间占比 |
| stopped_time_ratio | REAL | 停止时间占比 |
| market_status_changes | INTEGER | 行情状态变化次数 |
| market_algorithm | TEXT | 行情算法 |
| trading_statuses | TEXT (JSON) | 交易状态列表 |
| extra_metrics | TEXT (JSON) | 额外指标 |
| executed_at | DATETIME | 执行时间 |
| duration_ms | INTEGER | 执行耗时(毫秒) |
| status | TEXT | 状态 (running/success/failed) |

#### 3. 交易详情表 (trade_details)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INTEGER | 主键 (自增) |
| result_id | TEXT | 关联测试结果ID |
| trade_seq | INTEGER | 交易序号 |
| level | TEXT | 订单层级 (A1/A2/A3/A4) |
| entry_price | REAL | 入场价格 |
| exit_price | REAL | 出场价格 |
| trade_type | TEXT | 交易类型 (take_profit/stop_loss) |
| profit | REAL | 盈亏金额 |
| entry_time | DATETIME | 入场时间 |
| exit_time | DATETIME | 出场时间 |

### Requirement: API 接口设计

系统 SHALL 提供以下 API 接口：

#### 测试用例管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/test-cases | 获取测试用例列表 |
| POST | /api/test-cases | 创建测试用例 |
| GET | /api/test-cases/{id} | 获取测试用例详情 |
| PUT | /api/test-cases/{id} | 更新测试用例 |
| DELETE | /api/test-cases/{id} | 删除测试用例 |
| POST | /api/test-cases/{id}/run | 执行测试用例 |
| GET | /api/test-cases/{id}/export | 导出测试用例 (MD/JSON) |

#### 测试结果管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/test-results | 获取测试结果列表 |
| GET | /api/test-results/{id} | 获取测试结果详情 |
| GET | /api/test-results/{id}/trades | 获取交易详情 |
| GET | /api/test-results/{id}/export | 导出测试结果报告 |

#### 测试历史汇总

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/history | 获取测试历史汇总 |
| GET | /api/history/{symbol} | 获取指定标的测试历史 |
| GET | /api/history/export | 导出测试历史 (MD/CSV) |

#### 测试对比

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/compare | 对比多个测试结果 |
| GET | /api/compare/{id} | 获取对比结果 |
| POST | /api/compare/export | 导出对比报告 (MD) |
