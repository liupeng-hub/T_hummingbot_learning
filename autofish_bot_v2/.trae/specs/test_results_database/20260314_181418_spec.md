# 测试结果数据库管理方案 Spec

## Why

当前测试结果以 Markdown 文件形式分散存储，存在以下问题：
1. **查询困难**：无法快速查询和筛选历史测试结果
2. **对比不便**：难以进行多组测试结果的对比分析
3. **统计困难**：无法自动生成统计报表和趋势分析
4. **扩展性差**：新增测试类型需要修改多处代码

通过将测试参数和结果存储到 SQLite 数据库，可以实现：
- 统一的测试结果管理
- 便捷的查询和筛选
- 多维度对比分析
- 自动生成报告

## What Changes

- 创建 SQLite 数据库存储测试结果
- 设计统一的测试参数和结果表结构
- 实现测试结果写入和查询接口
- 支持从数据库生成 Markdown 报告
- 保持向后兼容，支持文件输出和数据库输出并行

## Impact

- Affected specs: test_manager.py, binance_backtest.py, market_aware_backtest.py
- Affected code: 新增 database/test_results.db

## ADDED Requirements

### Requirement: 测试结果数据库

系统 SHALL 提供 SQLite 数据库存储测试结果，支持以下功能：
- 存储测试计划和场景参数
- 存储测试结果指标
- 支持查询、筛选、对比
- 支持生成 Markdown 报告

### Requirement: 数据库操作接口

系统 SHALL 提供以下数据库操作接口：

```python
class TestResultsDB:
    def save_execution(self, execution: TestExecution) -> str:
        """保存测试执行记录，返回 execution_id"""
        
    def save_params(self, execution_id: str, params: dict) -> None:
        """保存测试参数"""
        
    def save_result(self, execution_id: str, result: TestResult) -> None:
        """保存测试结果"""
        
    def save_trade_details(self, execution_id: str, trades: List[Trade]) -> None:
        """保存交易明细"""
        
    def query_results(self, filters: dict) -> List[TestResult]:
        """查询测试结果"""
        
    def compare_results(self, execution_ids: List[str]) -> dict:
        """对比多个测试结果"""
        
    def generate_report(self, execution_id: str, format: str = 'md') -> str:
        """生成报告"""
        
    def get_history(self, symbol: str = None, test_type: str = None) -> List[dict]:
        """获取历史记录"""
```

### Requirement: 向后兼容

系统 SHALL 保持向后兼容：
- 单独执行脚本时，同时输出 MD 文件和写入数据库
- 测试框架执行时，写入数据库并可选生成 MD 报告
- 支持从数据库导出 MD 报告

### Requirement: 扩展性设计

系统 SHALL 支持扩展：
- extra_metrics 字段存储额外指标 (JSON 格式)
- params_json 字段存储场景特定参数
- 支持新增测试类型无需修改表结构

### Requirement: Web API 接口

系统 SHALL 提供 RESTful API 接口，支持 Web 前端访问：

```
GET  /api/test-plans              # 获取测试计划列表
GET  /api/test-plans/{plan_id}    # 获取测试计划详情
POST /api/test-plans              # 创建测试计划

GET  /api/test-executions         # 获取执行记录列表
GET  /api/test-executions/{id}    # 获取执行详情
POST /api/test-executions         # 创建执行记录

GET  /api/test-results            # 获取测试结果列表
GET  /api/test-results/{id}       # 获取测试结果详情
GET  /api/test-results/compare    # 对比多个测试结果

GET  /api/trade-details/{execution_id}  # 获取交易明细

GET  /api/history                 # 获取历史记录
GET  /api/statistics              # 获取统计数据

GET  /api/reports/{id}            # 获取报告 (支持 md/json 格式)
POST /api/reports/generate        # 生成报告
```

### Requirement: Web 管理界面

系统 SHALL 提供 Web 管理界面，支持以下功能：

**1. 测试计划管理**
- 测试计划列表展示
- 测试计划创建/编辑/删除
- 测试计划状态管理

**2. 测试执行管理**
- 执行记录列表展示
- 执行状态实时监控
- 执行日志查看

**3. 结果查看**
- 测试结果列表展示
- 单个结果详情查看
- 核心指标可视化图表

**4. 对比分析**
- 多组结果对比
- 参数差异高亮
- 指标对比图表

**5. 历史查询**
- 按标的/类型/时间筛选
- 历史趋势图表
- 数据导出
