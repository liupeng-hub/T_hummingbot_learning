# 测试计划数据迁移规范

## Why

当前数据库中的测试计划数据不完整，缺少关键字段（如 `execution`、 `test_parameters` 等），导致测试计划无法正常执行。需要从备份的 JSON 文件中重新导入完整的测试计划数据。

## What Changes

- 清空数据库中 `test_plans`、`test_scenarios` 表的数据
- 从备份的 JSON 文件中读取测试计划数据
- 将完整的测试计划数据导入到数据库中

## Impact

- Affected code: `database/test_results.db`
- Affected files: 备份的 JSON 文件（`out/test_old_bak/TP001_*.json`, `out/test_old_bak/TP002_*.json`, `out/test_old_bak/TP003_*.json`）

## ADDED Requirements

### Requirement: 数据迁移脚本

系统应提供数据迁移脚本，完成以下功能：

#### Scenario: 清空数据库
- **WHEN** 执行迁移脚本
- **THEN** 清空 `test_plans`、`test_scenarios` 表的数据

#### Scenario: 导入测试计划
- **WHEN** 从 JSON 文件读取测试计划数据
- **THEN** 将数据正确导入到 `test_plans`、`test_scenarios` 表中

#### Scenario: 验证导入结果
- **WHEN** 导入完成后
- **THEN** 验证数据完整性和正确性
