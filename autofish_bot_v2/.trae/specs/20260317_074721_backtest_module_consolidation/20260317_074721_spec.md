# 回测模块整合重构 Spec

## 背景
`market_aware_backtest.py` 继承自 `binance_backtest.py` 中的 `BacktestEngine` 类，并已完全覆盖了其功能。为了简化代码结构，减少文件依赖，需要将两个文件整合为一个。

## 变更内容
- 将 `binance_backtest.py` 中的 `BacktestEngine` 类和 `FlushFileHandler` 类整合到 `market_aware_backtest.py` 中
- 备份 `binance_backtest.py` 为 `binance_backtest_bak.py`
- 将 `market_aware_backtest.py` 重命名为 `binance_backtest.py`
- **备份说明**: 原 `binance_backtest.py` 文件备份为 `binance_backtest_bak.py`，保留不删除

## 影响范围
- 影响的功能: 回测模块
- 影响的代码: 
  - `binance_backtest.py` → 备份为 `binance_backtest_bak.py`
  - `market_aware_backtest.py` → 重命名为 `binance_backtest.py`
  - `test_manager.py` 中的命令行调用（无需修改，因为调用的是 `market_aware_backtest.py`）

## 新增需求

### 需求: 模块整合
系统 **应** 将 `BacktestEngine` 基类和 `MarketAwareBacktestEngine` 子类整合到同一个文件中。

#### 场景: 整合成功
- **当** 执行整合操作
- **则** 新的 `binance_backtest.py` 文件包含 `BacktestEngine` 类、`MarketAwareBacktestEngine` 类和 `FlushFileHandler` 类
- **且** 所有导入依赖正确

### 需求: 文件备份
系统 **应** 在修改前备份原始文件。

#### 场景: 备份成功
- **当** 执行备份操作
- **则** `binance_backtest.py` 被备份为 `binance_backtest_bak.py`
- **且** 原始文件内容完整保留

### 需求: 文件重命名
系统 **应** 将整合后的文件重命名为标准名称。

#### 场景: 重命名成功
- **当** 执行重命名操作
- **则** `market_aware_backtest.py` 被重命名为 `binance_backtest.py`
- **且** 文件内容包含所有必要的类和函数

## 修改需求

### 需求: 导入语句更新
整合后的文件 **应** 正确处理内部导入。

#### 场景: 内部类引用
- **当** `MarketAwareBacktestEngine` 继承 `BacktestEngine`
- **则** 不需要外部导入，两个类在同一文件中

## 移除需求

### 需求: 外部模块依赖
**原因**: 整合后不再需要从外部导入 `BacktestEngine`
**迁移方案**: 移除 `from binance_backtest import BacktestEngine` 语句
