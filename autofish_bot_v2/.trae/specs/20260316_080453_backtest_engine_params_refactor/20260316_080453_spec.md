# MarketAwareBacktestEngine 参数重构规格

## 背景
当前 `MarketAwareBacktestEngine` 接收两个参数（config, market_config），参数结构不够清晰，需要重构为更结构化的多参数格式，与 `autofish_extern_strategy.json` 的配置格式保持一致。

## 变更内容
- **破坏性变更**: `MarketAwareBacktestEngine.__init__` 参数签名变更
- 参数从 `(config, market_config)` 改为 `(amplitude, market, entry, timeout)`
- CLI 参数格式同步更新

## 影响范围
- 影响规格: market_aware_backtest
- 影响代码: 
  - `market_aware_backtest.py` (CLI 和 Engine 调用)
  - `MarketAwareBacktestEngine` 类

## 新增需求

### 需求: 新参数格式
`MarketAwareBacktestEngine` 构造函数 SHALL 接收以下四个参数：

```python
def __init__(
    self,
    amplitude: Dict,      # 振幅配置
    market: Dict,         # 行情算法配置
    entry: Dict,          # 入场价格策略配置
    timeout: Dict,        # 超时配置
):
```

#### 参数格式定义

**amplitude**: 振幅配置字典（不包含 symbol，symbol 由 CLI 必选参数指定）
```python
{
    "total_amount_quote": 5000,
    "leverage": 10,
    "decay_factor": 0.5,
    "max_entries": 4,
    "valid_amplitudes": [1, 2, 3, 4, 5, 6, 7, 8, 9],
    "weights": [0.0831, 0.2996, ...],
    "grid_spacing": 0.01,
    "exit_profit": 0.01,
    "stop_loss": 0.08
}
```

**注意**: `amplitude` 不包含 `symbol` 字段，`symbol` 由 CLI 必选参数 `--symbol` 指定。

**market**: 行情算法配置（只包含当前使用的算法参数）
```python
{
    "algorithm": "dual_thrust",           # 当前使用的算法
    "dual_thrust": {                       # dual_thrust 算法参数
        "n_days": 4,
        "k1": 0.4,
        "k2": 0.4,
        "k2_down_factor": 0.8,
        "down_confirm_days": 2,
        "cooldown_days": 1
    },
    "trading_statuses": ["ranging"]        # 允许交易的状态
}
```

**注意**: `market` 只包含当前使用的算法参数，未使用的算法参数不写入。

**entry**: 入场价格策略配置（只包含当前使用的策略参数）
```python
{
    "strategy": "atr",                     # 当前使用的策略
    "atr": {                               # atr 策略参数
        "atr_period": 14,
        "atr_multiplier": 0.5,
        "min_spacing": 0.005,
        "max_spacing": 0.03
    }
}
```

**注意**: `entry` 只包含当前使用的策略参数，未使用的策略参数不写入。

**timeout**: 超时配置
```python
{
    "a1_timeout_minutes": 0                # A1 入场超时时间
}
```

#### 场景: 成功创建引擎实例
- **当** 用户传入正确格式的四个参数
- **则** 引擎成功初始化，内部配置正确解析

## 修改需求

### 需求: CLI 参数格式
CLI 参数名称 SHALL 使用短字符串：

| 参数名 | 说明 |
|--------|------|
| `--symbol` | 交易对（必选） |
| `--date-range` | 时间范围（必选） |
| `--amplitude-params` | 振幅参数（JSON） |
| `--market-params` | 行情算法参数（JSON） |
| `--entry-params` | 入场价格策略参数（JSON） |
| `--timeout-params` | 超时参数（JSON） |

#### 场景: CLI 调用成功
- **当** 用户通过 CLI 传入 JSON 格式参数
- **则** 参数正确解析并传递给引擎

## 移除需求

### 需求: 旧参数格式
**原因**: 参数结构不清晰，与配置文件格式不一致
**迁移方案**: 将 `config` 和 `market_config` 拆分为四个独立参数
