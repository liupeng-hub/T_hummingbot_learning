# 资金算法问题澄清与优化

## 背景
用户对之前生成的复利资金算法评估报告提出了多个疑问和反馈，需要进一步分析和澄清入场资金逻辑、A1 超时重挂、提现阈值等问题，并根据用户的回测数据分析结果。

## 变更内容
- 澄清入场资金变化逻辑
- 分析 A1 超时重挂是否需要更新入场资金
- 调整提现阈值上限
- 统一 EntryCapitalStrategy 处理
- 分析权重分配机制
- 澄清 entry_capital 和 entry_total_capital 用途
- 更新文档文件名增加时间信息

## 影响范围
- 涉及代码: autofish_core.py, binance_backtest.py
- 涉及文档: docs/capital_algorithm_evaluation.md, docs/trading_algorithm.md

## 新增需求

### 需求：入场资金逻辑澄清
系统应明确入场资金变化的时机：
- A1 挂单成交但未达止盈止损时，A2 使用 A1 的入场资金
- A2 成交但未止盈止损时，入场资金不变
- A3 止盈或止损后，入场资金更新，重新下发 A3 使用更新后的入场资金

#### 场景：入场资金不变
- **当** A1/A2 成交但未止盈止损时
- **则** 后续订单使用相同入场资金

#### 场景：入场资金更新
- **当** 任意订单止盈或止损时
- **则** 入场资金更新为当前总资金

### 需求：A1 超时重挂分析
系统应分析 A1 超时重挂是否需要更新入场资金：
- 如果没有其他订单成交，入场资金不变
- 如果有止盈止损导致资金变化，需要更新入场资金

### 需求：提现阈值调整
系统应将提现阈值上限从 999 调整为 99999999，支持更大范围的复利策略。

### 需求：统一 EntryCapitalStrategy
系统应统一使用 EntryCapitalStrategy 处理入场资金计算，消除重复逻辑。

### 需求：权重分配机制分析
系统应分析当前权重分配机制是否合理，并提供改进建议。

### 需求：entry_capital 和 entry_total_capital 澄清
系统应明确这两个字段的用途：
- entry_capital: 入场时的交易资金
- entry_total_capital: 入场时的总资金（交易资金 + 利润池）

### 需求：文档文件名增加时间信息
系统应将文档文件名增加时间信息前缀。
