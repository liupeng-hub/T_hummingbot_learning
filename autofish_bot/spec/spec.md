# Autofish V1 算法系统规格说明

## 一、系统概述

### 1.1 算法定义

**Autofish V1** 是一个**基于振幅概率统计的加权网格震荡交易系统**，核心特点：

- **数据驱动**：通过历史K线数据分析价格波动幅度的概率分布
- **加权网格**：根据不同振幅区间的预期收益计算资金分配权重
- **链式挂单**：在价格下跌时逐层建仓，每层订单独立止盈止损
- **震荡交易**：适合震荡行情，在价格波动中获取收益

### 1.2 核心思想

```
振幅 = (high - low) / open × 100

预期收益 = 振幅 × 杠杆 × 概率

权重 = 振幅 × 概率^(1/d)  // d 为衰减因子
```

**关键优化**：剔除负收益区间（≥10%振幅会触发强平），只保留正收益区间参与交易。

## 二、振幅权重计算（核心算法）

### 2.1 振幅定义

```
振幅 = (K线最高价 - K线最低价) / K线开盘价 × 100
```

### 2.2 概率统计

通过历史K线数据统计各振幅区间的出现频率：

| 振幅区间 | BTCUSDT 概率 | ETHUSDT 概率 |
|---------|-------------|-------------|
| 1% | 18.2% | 22.1% |
| 2% | 24.4% | 28.3% |
| 3% | 20.4% | 19.7% |
| 4% | 11.6% | 12.5% |
| 5% | 8.2% | 7.8% |

### 2.3 预期收益计算

```
预期收益 = 振幅 × 杠杆 × 概率
```

**示例（BTCUSDT，10倍杠杆）**：
- 1%振幅：预期收益 = 1% × 10 × 0.182 = 1.82%
- 2%振幅：预期收益 = 2% × 10 × 0.244 = 4.88%
- 3%振幅：预期收益 = 3% × 10 × 0.204 = 6.12%
- 4%振幅：预期收益 = 4% × 10 × 0.116 = 4.64%

### 2.4 权重计算公式

```
权重 = 振幅 × 概率^(1/d)

其中 d 为衰减因子：
- d=0.5（激进策略）：幂次β=2，权重集中在前几层
- d=1.0（保守策略）：幂次β=1，权重分布均匀
```

### 2.5 权重分配示例

**BTCUSDT 权重分配（d=0.5，max_entries=4）**：

| 层级 | 振幅 | 权重 | 资金（1200 USDT） |
|------|------|------|------------------|
| A1 | 1% | 8.40% | 100.8 USDT |
| A2 | 2% | 30.18% | 362.16 USDT |
| A3 | 3% | 31.65% | 379.8 USDT |
| A4 | 4% | 13.64% | 163.68 USDT |

## 三、链式挂单过程

### 3.1 初始化流程

```
1. 获取当前价格（基准价格）
2. 创建 A1 订单：
   - 入场价 = 基准价 × (1 - 1%)
   - 止盈价 = 入场价 × (1 + 1%)
   - 止损价 = 入场价 × (1 - 8%)
   - 资金 = 总资金 × A1权重
3. 下 A1 入场单（LIMIT 限价单）
```

### 3.2 成交后流程（链式）

```
A1 成交：
  ↓
1. 下止盈止损条件单（Algo Order）
   - 止盈单：TAKE_PROFIT_MARKET，触发价=止盈价
   - 止损单：STOP_MARKET，触发价=止损价
  ↓
2. 创建 A2 订单：
   - 入场价 = A1入场价 × (1 - 1%)
   - 资金 = 总资金 × A2权重
  ↓
3. 下 A2 入场单
```

### 3.3 止盈触发流程

```
A1 止盈触发：
  ↓
1. 计算盈利
2. 取消 A1 止损单
3. 取消 A2 入场单（下一级挂单）
4. 删除 A2 本地记录
5. 重新创建 A1 入场单（基于当前价格）
```

### 3.4 止损触发流程

```
A1 止损触发：
  ↓
1. 计算亏损
2. 取消 A1 止盈单
3. 取消 A2 入场单（下一级挂单）
4. 删除 A2 本地记录
5. 重新创建 A1 入场单（基于当前价格）
```

### 3.5 完整链式挂单示例

```
初始状态：
  A1: pending（入场价 99000）

价格下跌到 99000：
  A1: filled（成交价 99000）
  A1-TP: pending（触发价 99990）
  A1-SL: pending（触发价 91080）
  A2: pending（入场价 98010）

价格继续下跌到 98010：
  A2: filled（成交价 98010）
  A2-TP: pending（触发价 98990）
  A2-SL: pending（触发价 90169）
  A3: pending（入场价 97030）

价格反弹到 99990：
  A1-TP: triggered → A1 closed（止盈）
  取消 A1-SL
  取消 A2 入场单
  删除 A2 本地记录
  重新创建 A1（入场价 98990）
```

## 四、订单状态管理

### 4.1 订单状态流转

```
pending（挂单中）
    ↓ 成交
filled（已成交）
    ↓ 止盈/止损触发
closed（已平仓）

pending → cancelled（用户取消）
pending → expired（订单过期）
```

### 4.2 订单数据结构

```python
@dataclass
class Order:
    level: int                          # 层级（A1, A2, A3, A4）
    entry_price: Decimal                # 入场价
    quantity: Decimal                   # 数量
    stake_amount: Decimal               # 投入金额
    take_profit_price: Decimal          # 止盈价
    stop_loss_price: Decimal            # 止损价
    state: str = "pending"              # 状态
    order_id: Optional[int] = None      # 主订单ID
    tp_order_id: Optional[int] = None   # 止盈条件单ID
    sl_order_id: Optional[int] = None   # 止损条件单ID
```

### 4.3 订单状态说明

| 状态 | 说明 | order_id | tp_order_id | sl_order_id |
|------|------|----------|-------------|-------------|
| pending | 挂单中 | ✅ 有 | ❌ None | ❌ None |
| filled | 已成交 | ✅ 有 | ✅ 有 | ✅ 有 |
| closed | 已平仓 | ✅ 有 | ❌ None | ❌ None |
| cancelled | 已取消 | ✅ 有 | ❌ None | ❌ None |

## 五、价格计算逻辑

### 5.1 入场价计算

```python
entry_price = base_price * (1 - grid_spacing)
```

- `base_price`：基准价格（当前价格或上一级入场价）
- `grid_spacing`：网格间距（默认 1%）

### 5.2 止盈价计算

```python
take_profit_price = entry_price * (1 + exit_profit)
```

- `exit_profit`：止盈比例（默认 1%）

### 5.3 止损价计算

```python
stop_loss_price = entry_price * (1 - stop_loss)
```

- `stop_loss`：止损比例（默认 8%）

### 5.4 数量计算

```python
quantity = stake_amount / entry_price
```

- `stake_amount`：投入金额 = 总资金 × 权重

## 六、盈亏计算

### 6.1 盈亏公式

```python
profit = stake_amount * (close_price - entry_price) / entry_price * leverage
```

### 6.2 盈亏示例

**止盈场景（10倍杠杆）**：
- 入场价：99000 USDT
- 止盈价：99990 USDT
- 投入：100.8 USDT
- 盈利 = 100.8 × (99990 - 99000) / 99000 × 10 = **10.08 USDT**（+10%）

**止损场景（10倍杠杆）**：
- 入场价：99000 USDT
- 止损价：91080 USDT
- 投入：100.8 USDT
- 亏损 = 100.8 × (91080 - 99000) / 99000 × 10 = **-80.64 USDT**（-80%）

### 6.3 盈亏比分析

- 止盈收益：+10%（1% × 10倍杠杆）
- 止损亏损：-80%（8% × 10倍杠杆）
- 盈亏比：1:8
- 需要胜率：至少 8/(8+1) = 88.9% 才能盈利

## 七、订单恢复和同步机制

### 7.1 恢复流程

```
1. 加载本地状态文件
2. 查询 Binance 实际状态
3. 同步本地状态与 Binance 实际状态
4. 补充缺失的止盈止损单
5. 调整订单级别
```

### 7.2 状态同步逻辑

| 本地状态 | Binance 状态 | 处理 |
|---------|-------------|------|
| pending | FILLED | 更新为 filled |
| pending | CANCELED | 删除本地记录 |
| pending | NEW | 无需处理 |
| filled | 无止盈止损单 | 补单 |
| filled | 无仓位 | 判断止盈/止损已成交，删除 |

### 7.3 补单机制

- **止盈单缺失**：检查当前价是否已超过止盈价，调整止盈价后下单
- **止损单缺失**：检查当前价是否已跌破止损价，市价止损或下条件单

## 八、风险控制

### 8.1 止损设置

- 默认止损：8%
- 止盈止损比：1:8
- 需要较高胜率才能盈利

### 8.2 最大层级限制

- 默认最大层级：4层
- 前4层权重合计：83.87%
- 避免过度加仓

### 8.3 强平风险规避

- 剔除≥10%振幅区间（10倍杠杆下会触发强平）
- 只保留正收益区间参与交易

## 十一、适用场景

### 11.1 适合的市场

- **震荡行情**：价格在一定区间内波动
- **中低波动率**：振幅在 1-4% 区间
- **流动性好**：订单能及时成交

### 11.2 不适合的市场

- **单边趋势**：持续上涨或下跌
- **高波动率**：振幅超过 8%
- **流动性差**：订单成交困难

## 十二、改进方向

### 12.1 短期改进

1. **动态止损**：根据波动率动态调整止损比例
2. **仓位控制**：根据市场趋势动态调整最大层级
3. **多时间框架**：结合多周期分析优化入场时机

### 12.2 长期改进

1. **机器学习**：使用 ML 优化权重分配
2. **多品种组合**：相关性对冲降低风险
3. **自适应参数**：根据市场状态自动调整参数
