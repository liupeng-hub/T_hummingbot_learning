# Autofish V1 检查清单

## 一、核心算法检查

### 1.1 振幅权重计算
- [ ] 振幅定义是否正确：`(high - low) / open × 100`
- [ ] 概率统计是否基于足够的历史数据（≥1000根K线）
- [ ] 预期收益计算是否正确：`振幅 × 杠杆 × 概率`
- [ ] 权重计算公式是否正确：`振幅 × 概率^(1/d)`
- [ ] 权重是否归一化（总和为1）
- [ ] 是否剔除了负收益区间（≥10%振幅）

### 1.2 链式挂单检查
- [ ] 初始化时是否只创建 A1 订单
- [ ] 成交后是否先下止盈止损单，再创建下一级订单
- [ ] 止盈触发后是否取消了止损单和下一级挂单
- [ ] 止损触发后是否取消了止盈单和下一级挂单
- [ ] 止盈/止损后是否重新创建了该层级订单

---

## 二、订单状态检查

### 2.1 状态流转
- [ ] pending → filled：成交后状态是否正确更新
- [ ] filled → closed：止盈/止损后状态是否正确更新
- [ ] pending → cancelled：取消后状态是否正确更新
- [ ] 订单 ID 是否正确记录

### 2.2 数据完整性
- [ ] pending 状态：order_id 是否有值，tp_order_id 和 sl_order_id 是否为 None
- [ ] filled 状态：order_id、tp_order_id、sl_order_id 是否都有值
- [ ] closed 状态：order_id 是否有值，tp_order_id 和 sl_order_id 是否为 None

---

## 三、价格计算检查

### 3.1 入场价
- [ ] 入场价是否正确：`base_price × (1 - grid_spacing)`
- [ ] A2 入场价是否基于 A1 入场价计算
- [ ] 入场价是否保留足够精度

### 3.2 止盈价
- [ ] 止盈价是否正确：`entry_price × (1 + exit_profit)`
- [ ] 止盈比例是否符合配置

### 3.3 止损价
- [ ] 止损价是否正确：`entry_price × (1 - stop_loss)`
- [ ] 止损比例是否符合配置

### 3.4 数量计算
- [ ] 数量是否正确：`stake_amount / entry_price`
- [ ] 数量是否符合交易所最小交易量要求

---

## 四、盈亏计算检查

### 4.1 公式检查
- [ ] 盈亏公式是否正确：`stake_amount × (close_price - entry_price) / entry_price × leverage`
- [ ] 杠杆是否正确应用
- [ ] 盈亏符号是否正确（正为盈利，负为亏损）

### 4.2 统计检查
- [ ] 总交易次数是否正确统计
- [ ] 盈利交易次数是否正确统计
- [ ] 亏损交易次数是否正确统计
- [ ] 总盈利金额是否正确累计
- [ ] 总亏损金额是否正确累计
